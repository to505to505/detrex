## Deformable Attention по иллюстрации (Figure 2)

Эта схема иллюстрирует, как работает модуль **Deformable Attention** для одного элемента запроса (`Query Feature z_q`) и одной входной карты признаков (`Input Feature Map x`). Здесь показан пример с 3-мя "головами" внимания (Head 1, Head 2, Head 3) и 3-мя точками сэмплинга (`K=3`) на каждую голову.

**Входные данные:**

1.  **`Query Feature z_q` (Признак Запроса):** Это векторное представление текущего запроса. Например, в декодере это может быть признак одного из объектных запросов (object query), а в энкодере – признак пикселя с карты признаков.
2.  **`Reference Point p_q` (Опорная Точка):** Это 2D-координаты (`p_qx`, `p_qy`) на входной карте признаков, связанные с текущим запросом `z_q`. Эта точка служит "якорем" или начальной точкой для поиска релевантной информации.
3.  **`Input Feature Map x` (Входная Карта Признаков):** Это многоканальная карта признаков, из которой мы хотим извлечь информацию (например, выход одного из слоев CNN).

**Процесс:**

1.  **Генерация Смещений и Весов (Параллельно):**
    *   **`Sampling Offsets {Δp_mqk}` (Смещения для Сэмплинга):**
        *   Признак запроса `z_q` подается на **линейный слой (Linear)**.
        *   Выход этого слоя интерпретируется как набор 2D-смещений `Δp_mqk`. Для каждой из `M` голов внимания (здесь `M=3`) и для каждой из `K` точек сэмплинга (здесь `K=3`) предсказывается свое 2D-смещение.
        *   На схеме для каждой головы (Head 1, 2, 3) показаны 3 стрелочки – это и есть 3 предсказанных 2D-смещения относительно опорной точки `p_q`. Эти смещения определяют, *где* на карте признаков `x` нужно взять информацию.
    *   **`Attention Weights {A_mqk}` (Веса Внимания):**
        *   Признак запроса `z_q` также (параллельно) подается на **другой линейный слой (Linear)**, за которым следует функция **Softmax**.
        *   Выход этого блока – это набор весов внимания `A_mqk`. Для каждой из `M` голов и для каждой из `K` точек сэмплинга предсказывается свой вес. Эти веса показывают, *насколько важна* информация, взятая из каждой сэмплированной точки.
        *   На схеме для каждой головы (Head 1, 2, 3) показаны 3 значения весов (например, для Head 1: 0.5, 0.3, 0.2), сумма которых для каждой головы равна 1 (из-за Softmax).

2.  **Сэмплирование Значений (`Values {W'_m x}`):**
    *   Входная карта признаков `x` проходит через **линейный слой (Linear)**. Это делается для того, чтобы спроецировать признаки `x` в пространство "значений" (values), возможно, с другой размерностью. Результат обозначен как `W'_m x` (хотя на схеме это общий блок для всех голов, в реальности каждая голова может иметь свою проекцию `W'_m`).
    *   Теперь для каждой головы внимания (Head 1, 2, 3) происходит следующее:
        *   Берется опорная точка `p_q`.
        *   К ней прибавляются предсказанные ранее 2D-смещения `Δp_mqk` для данной головы. Получаются `K` (здесь 3) абсолютных координат на карте признаков `W'_m x`.
        *   В этих `K` вычисленных координатах с помощью **билинейной интерполяции** (поскольку смещения могут быть нецелочисленными) сэмплируются векторы признаков.
        *   На схеме это показано как маленькие квадратики, извлекаемые из карт `W'_m x` для каждой головы в местах, указанных смещениями.

3.  **Агрегация Взвешенных Значений:**
    *   Для каждой головы внимания (Head 1, 2, 3):
        *   Берутся `K` сэмплированных векторов признаков (из шага 2).
        *   Берутся `K` соответствующих весов внимания `A_mqk` (из шага 1).
        *   Выполняется **взвешенное суммирование**: каждый сэмплированный вектор признаков умножается на свой вес внимания, и результаты суммируются.
        *   `Aggregated_Value_m = ∑_k (A_mqk * Sampled_Value_mk)`
        *   На схеме это показано блоками "Aggregate", которые для каждой головы объединяют 3 взвешенных сэмплированных значения в один результирующий вектор для данной головы. Получаются `M` (здесь 3) таких агрегированных векторов – `Aggregated Sampled Values`.

4.  **Финальный Выход:**
    *   `M` агрегированных векторов значений от каждой головы конкатенируются (или суммируются, в зависимости от реализации multi-head attention).
    *   Этот объединенный вектор затем проходит через **финальный линейный слой (Linear)**, чтобы получить итоговый выходной вектор `Output` для данного запроса `z_q`.

**Ключевые идеи, видимые на схеме:**

*   **Адаптивность:** Местоположения, из которых извлекается информация (смещения), и важность этой информации (веса) *динамически предсказываются* на основе текущего запроса `z_q`.
*   **Разреженность (Sparsity):** Вместо того чтобы обрабатывать всю карту признаков `x`, внимание фокусируется только на небольшом (`K`) количестве точек для каждой головы.
*   **Multi-Head:** Использование нескольких "голов" позволяет модели одновременно фокусироваться на разных паттернах или частях информации.
*   **Отсутствие глобальной матрицы схожести:** В отличие от стандартного attention, здесь нет явного вычисления матрицы схожести между запросом и всеми возможными ключами на карте признаков.

Этот механизм позволяет Deformable Attention быть более эффективным и быстрее сходиться, особенно при работе с большими картами признаков, что важно для задач, таких как детектирование объектов.
