### Общая архитектура потока данных
Метод `forward` реализует полный проход от признаков энкодера до финальных предсказаний. Процесс делится на следующие этапы:

1.  **Подготовка и унификация мультимасштабных признаков.**
2.  **Инициализация векторов-запросов (queries)**: контентной и пространственной части.
3.  **Аугментация запросов для Denoising (DN)** (только на этапе обучения).
4.  **Исполнение трансформер-декодера** с использованием `MultiScaleDeformableAttention` для итеративного уточнения запросов.
5.  **Генерация предсказаний** с помощью специализированных "голов" (prediction heads).
6.  **Формирование и возврат выходного словаря.**

---

## Этап 1: Подготовка мультимасштабных признаков

*   **Вход:** `x` — список тензоров, представляющих карты признаков с нескольких уровней (`L`) энкодера. Каждый тензор имеет размерность `[Batch, Channels_l, H_l, W_l]`.
*   **Цель:** Преобразовать набор 2D-карт признаков в единую 1D-последовательность для обработки трансформером.
*   **Реализация:**
    1.  Производится итерация по картам признаков в обратном порядке (от `L-1` до `0`), что позволяет обрабатывать сначала семантически богатые признаки низкого разрешения.
    2.  **Проекция каналов:** Каждая карта `x[idx]` проходит через соответствующий слой `self.input_proj[idx]` (свёртка 1x1) для унификации числа каналов до `hidden_dim`.
    3.  **Выравнивание и Транспонирование:** Пространственные измерения каждой карты `[B, C, H, W]` выравниваются в `[B, C, H*W]` (`.flatten(2)`) и транспонируются в `[B, H*W, C]` (`.transpose(1, 2)`).
    4.  **Конкатенация:** Все полученные 1D-последовательности конкатенируются вдоль измерения `SeqLen` (`dim=1`), формируя единый тензор `src_flatten` размером `[B, Sum(H_l*W_l), C]`.
    5.  **Генерация метаданных:** Одновременно создаются тензоры `spatial_shapes` (содержит `[H_l, W_l]` для каждого уровня) и `level_start_index` (содержит стартовый индекс каждого уровня в `src_flatten`). Эти метаданные критически важны для `MultiScaleDeformableAttention`.

---

## Этап 2: Инициализация запросов (Content & Spatial Queries)

### A. Двухстадийный режим (`two_stage = True`)

1.  **Генерация предложений (Proposals):** `gen_encoder_output_proposals` создает плотный набор bounding box'ов, по одному для каждой пространственной позиции в `src_flatten`.
2.  **Начальные предсказания:** К `output_memory` (обработанные признаки энкодера) применяются начальные "головы" для предсказания классов и координат для всех предложений.
3.  **Выбор Top-K:** С помощью `torch.topk` отбираются `num_queries` предложений с наивысшими баллами уверенности в классе.
4.  **Инициализация запросов:** `torch.gather` использует индексы top-k для выборки соответствующих векторов признаков и координат.
    *   **`tgt` (Content Query):** Векторы признаков top-k предложений.
    *   **`refpoint_embed` (Spatial Query):** Координаты bounding box'ов top-k предложений.

### B. Одностадийный режим (`two_stage = False`)

Инициализация не зависит от входных данных:
*   **`tgt`:** Инициализируется из `self.query_feat` — обучаемого `nn.Embedding`.
*   **`refpoint_embed`:** Инициализируется из `self.query_embed` — другого обучаемого `nn.Embedding`.

---

## Этап 3: Denoising (DN) Query Augmentation (Только при обучении)

1.  **`prepare_for_dn`:** Функция берёт истинные `targets`, создаёт их зашумлённые копии (добавляя шум к координатам и классам) и преобразует в эмбеддинги `input_query_label` и `input_query_bbox`.
2.  **Конкатенация:** Зашумлённые запросы конкатенируются с основными (из Этапа 2), размещаясь в начале последовательности.
3.  **Маска внимания (`tgt_mask`):** Генерируется маска, предотвращающая "общение" между зашумлёнными и основными запросами.

---

## Этап 4: Исполнение Трансформер-декодера и `MultiScaleDeformableAttention`

Это центральный вычислительный блок. Декодер итеративно уточняет запросы. Ключевым механизмом внутри каждого слоя является **`MultiScaleDeformableAttention`**.

*   **Проблема, которую решает `MultiScaleDeformableAttention`**: Обычный `Deformable Attention` эффективно работает с одной картой признаков. Однако у нас есть `src_flatten`, который является конкатенацией **нескольких** карт разного масштаба. Механизм должен уметь эффективно сэмплировать признаки со всех этих уровней одновременно.

*   **Принцип работы `MultiScaleDeformableAttention`**:
    1.  **Мультимасштабные опорные точки (`reference_points`):** В отличие от простого Deformable Attention, здесь каждый запрос имеет опорную точку **для каждого уровня признаков**. Тензор `reference_points` имеет размер `[Batch, NumQueries, NumLevels, 4]`. Это позволяет запросу иметь "якорь" одновременно и на крупномасштабной, и на мелкомасштабной карте.
    2.  **Мультимасштабные смещения (`sampling_offsets`):** Полносвязный слой, принимающий на вход контент запроса (`query`), предсказывает смещения `(Δx, Δy)` также **для каждого уровня признаков**. Его выход имеет размерность `[B, Q, Heads, Levels, Points, 2]`. Это позволяет модели выучивать разную геометрию "осмотра" для разных масштабов. Например, для опознания человека на грубой карте (низкое разрешение) модель может посмотреть на центр объекта, а на детальной карте — на голову и плечи.
    3.  **Сэмплирование по разным уровням:** Для каждого запроса механизм выполняет следующее:
        *   Берёт опорную точку для уровня `l`.
        *   Прибавляет к ней предсказанное смещение для уровня `l`.
        *   Используя `level_start_index` и `spatial_shapes`, находит соответствующий фрагмент в `src_flatten` и сэмплирует оттуда вектор признака с помощью билинейной интерполяции.
    4.  **Агрегация информации:** Признаки, собранные со всех `L` уровней для `N_points` точек, агрегируются в единый вектор через взвешенную сумму. Веса внимания также предсказываются для каждой точки на каждом уровне.
    5.  **Обновление запроса:** Этот агрегированный вектор, содержащий мультимасштабную визуальную информацию, используется для обновления контентного вектора запроса `tgt`.

*   **Выход декодера:**
    *   `hs`: `[NumLayers, NumQueries, B, C]` — иерархия уточнённых контентных эмбеддингов.
    *   `references`: `[NumLayers, NumQueries, B, 4]` — иерархия уточнённых пространственных координат.

---

## Этап 5: Генерация предсказаний

Выход `hs` последнего слоя декодера подаётся на три независимые "головы":

1.  **Классификация (`pred_logits`):** `hs` проецируется через `self.class_embed` (`nn.Linear`) в пространство классов.
2.  **Маски (`pred_masks`):** `hs` проходит через MLP `self.mask_embed`, генерируя компактный вектор-представление маски. Этот вектор умножается (`torch.einsum`) на карту признаков высокого разрешения `mask_features` для получения пространственной маски.
3.  **Ограничивающие рамки (`pred_boxes`):** `hs` проходит через MLP `self.decoder.bbox_embed`, предсказывая **смещение** `(Δcx, Δcy, Δw, Δh)`. Это смещение применяется к `references` из предыдущего слоя, реализуя итеративное уточнение.

---

## Этап 6: Формирование выходного словаря

1.  **Пост-обработка DN:** Если необходимо, `dn_post_process` разделяет предсказания для DN loss и matching loss.
2.  **Сборка `out`:**
    *   **Основные предсказания:** Ключи `'pred_logits'`, `'pred_masks'`, `'pred_boxes'` заполняются результатами от **последнего** слоя декодера.
    *   **Вспомогательные выходы (`'aux_outputs'`):** Создаётся список словарей с предсказаниями от **всех промежуточных** слоёв для реализации "глубокого обучения" (deep supervision).
3.  Возвращается словарь `out` и, если применимо, `mask_dict`.
